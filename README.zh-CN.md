## 📖 项目描述

**Dream Creator** 是一个由 **12 个专业 AI 代理**组成的**虚拟开发团队**。不同于单一 AI 助手，它模拟真实开发团队的协作模式，帮助你从零创建应用程序。

无论你是编程新手还是资深开发者，Dream Creator 都能帮助你：
- 🚀 **从零构建应用** - AI 交互式引导每一步
- 📝 **文档化现有项目** - 自动分析和生成文档
- 🔄 **迭代开发功能** - 自动化代码审查和测试
- 🧠 **架构方案建议** - AI 专家提供技术方案

## 简介

Dream Creator 不仅仅是一个工具——它是你的**虚拟开发团队**。当你调用 `/dream-creator` 时，你得到的不是一个单一的 AI 助手，而是一个由专业代理组成的完整团队，共同协作将你的想法变为现实。

### Dream Creator 的独特之处

与传统的人工智能编码助手不同，Dream Creator 实现了**多代理系统**，不同的专业代理之间可以协作、沟通和相互交接任务——就像真实的开发团队一样。

## 🚀 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/Xianyu33666/Dream-Creator.git
cd Dream-Creator

# 运行安装脚本
# Linux/macOS
chmod +x install.sh
./install.sh

# Windows
.\install.ps1
```

### 使用

在 Claude Code 中调用：

```
/dream-creator
```

## 🔄 代理如何协作

Dream Creator 实现了智能的**代理间通信**。以下是一些关键场景：

### 场景 1: 事故响应 (运维 → 开发 → 审查 → 测试 → 运维)

```
🐛 运维发现错误 → 💻 开发修复 → 🔍 审查批准 → 🧪 测试通过 → 🚀 运维部署
```

### 场景 2: 架构咨询 (开发 ↔ 架构师)

```
💻 开发有疑问 → 🏗️ 架构师提供方案 → 💻 开发实现
```

### 场景 3: 需求澄清 (开发 → 产品 → 用户 → 产品 → 开发)

```
💻 需求不明确 → 👤 产品询问用户 → 👤 产品澄清 → 💻 开发实现
```

### 场景 4: Bug 报告 (测试 → 开发)

```
🧪 测试发现 bug → 💻 开发修复 → 🔍 审查重新检查 → 🧪 测试验证
```

### 场景 5: 知识积累 (开发 → FAQ)

```
💻 解决问题 → 📚 FAQ 代理记录 → 未来: 快速获得答案
```

## 🔐 通信协议

所有代理都使用标准化协议进行通信：

```markdown
## 代理通信

**发送方**: [源代理]
**接收方**: [目标代理]
**类型**: [请求/响应/升级/反馈/通知]
**优先级**: [紧急/高/中/低]

---
### 主题: [简述]

### 详情
[详细信息]

### 需要的操作
[接收者需要做什么]
```

### 优先级级别

| 优先级 | 使用场景 | 响应时间 |
|--------|---------|---------|
| **紧急** | 生产环境 bug、安全问题 | 即时 |
| **高** | 阻塞性问题 | < 1 小时 |
| **中** | 问题、澄清 | < 4 小时 |
| **低** | 知识分享 | < 24 小时 |

## 💡 为什么选择多代理？

传统的人工智能助手独自工作。Dream Creator 作为一个**团队**工作：

1. **专业化**: 每个代理都是其领域的专家
2. **协作**: 代理之间相互沟通和交接任务
3. **升级机制**: 问题流向正确的专家
4. **知识共享**: 记录解决方案供将来参考
5. **质量保证**: 多个代理审查和测试每个变更
