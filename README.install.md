**AI 应用孵化器** — 一个由 12 个专业 AI 代理组成的虚拟开发团队，帮助你通过交互式对话创建应用程序。

## 核心特性

### 🤖 虚拟开发团队
Dream Creator 不仅仅是一个工具——它是你的**虚拟开发团队**。由 **12 个专业代理**组成，模拟真实开发团队的协作模式。

| 代理 | 角色 |
|------|------|
| 👤 产品经理 | 主编，与用户沟通 |
| 🏗️ 架构师 | 技术方案设计 |
| 💻 前端开发 | UI 实现 |
| ⚙️ 后端开发 | API 开发 |
| 🧪 测试工程师 | 质量保证 |
| 🔍 代码审查 | 代码审核 |
| 🚀 运维工程师 | 部署上线 |
| 🔧 环境设置 | 环境配置 |
| 🐛 调试器 | 问题诊断 |
| 📝 技术文档 | 文档编写 |
| 📚 FAQ 代理 | 知识库管理 |
| 👋 欢迎代理 | 首次引导 |

### 🔄 智能代理协作
- **事故响应**: 运维 → 开发 → 审查 → 测试 → 部署
- **架构咨询**: 开发 ↔ 架构师
- **需求澄清**: 开发 → 产品 → 用户 → 产品 → 开发
- **知识积累**: 开发 → FAQ → 未来快速解决

## 功能特性

- **🤖 多代理系统**: 12 个专业代理协同工作
- **💬 交互式需求收集**: 通过 2-5 轮对话了解用户需求
- **🧠 智能框架推荐**: AI 驱动的技术栈推荐
- **🔧 自动环境搭建**: 自动安装依赖和配置开发环境
- **🔍 代码审核集成**: 自动进行代码审查和测试
- **🔄 迭代开发**: 支持功能的持续迭代开发
- **📚 知识库**: FAQ 系统从问题中学习
- **🌐 多语言支持**: 支持中文、英文、繁体中文

## 安装方式

### 方式一：npx 一键安装（推荐）

```bash
npx dream-creator
```

### 方式二：全局安装

```bash
npm install -g dream-creator
dream-creator
```

### 方式三：手动安装

#### Windows PowerShell

```powershell
.\install.ps1
```

#### Linux / macOS / WSL

```bash
chmod +x install.sh
./install.sh
```

## 使用方法

安装完成后，在支持 Skills 的 AI 工具中输入：

```
/dream-creator
```

## 支持的工具

- Claude Code
- Cursor
- OpenCode

## 系统要求

- Node.js 14.0 或更高版本（用于 npx/npm 安装）
- 至少一个支持的 AI 编程工具
