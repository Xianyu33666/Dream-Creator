## 簡介

Dream Creator 不僅僅是一個工具——它是你的**虛擬開發團隊**。當你呼叫 `/dream-creator` 時，你得到的不是一個單一的 AI 助手，而是由專業代理組成的完整團隊，共同協作將你的想法變為現實。

### Dream Creator 的獨特之處

與傳統的人工智慧編碼助手不同，Dream Creator 實現了**多代理系統**，不同的專業代理之間可以協作、溝通和相互交接任務——就像真實的開發團隊一樣。

## 🚀 快速開始

### 安裝

Dream Creator 支援多種安裝方式，選擇最適合你的方式：

#### 方式 1: npm (推薦)

透過 npm 全域安裝（需要 [Node.js](https://nodejs.org/)）：

```bash
# 全域安裝
npm install -g dream-creator

# 或不安裝直接使用 npx
npx dream-creator
```

#### 方式 2: 複製並安裝

複製儲存庫並執行安裝腳本：

```bash
# 複製儲存庫
git clone https://github.com/Xianyu33666/Dream-Creator.git
cd Dream-Creator

# 執行安裝腳本（需要 Node.js）
# Linux/macOS
chmod +x install.sh
./install.sh

# Windows PowerShell
.\install.ps1
```

#### 方式 3: 手動安裝

將技能檔案複製到你的 AI 工具對應目錄：

| 工具 | 安裝路徑 |
|------|----------|
| **Claude Code** (個人級) | `~/.claude/skills/dream-creator/` |
| **Cursor** (使用者級) | `~/.cursor/skills/dream-creator/` |
| **OpenCode** (全域設定) | `~/.config/opencode/skills/dream-creator/` |

### 更新

更新到最新版本：

```bash
# 如果是透過 npm 安裝
npm update -g dream-creator

# 如果是從儲存庫複製
./install.sh -Force      # Linux/macOS
.\install.ps1 -Force     # Windows
```

### 使用

在 Claude Code 中呼叫：

```
/dream-creator
```

## 🔄 代理如何協作

Dream Creator 實現了智慧的**代理間通信**。以下是一些關鍵場景：

### 場景 1: 事故回應 (維運 → 開發 → 審查 → 測試 → 維運)

```
🐛 維運發現錯誤 → 💻 開發修復 → 🔍 審查批准 → 🧪 測試通過 → 🚀 維運部署
```

### 場景 2: 架構諮詢 (開發 ↔ 架構師)

```
💻 開發有疑問 → 🏗️ 架構師提供方案 → 💻 開發實現
```

### 場景 3: 需求澄清 (開發 → 產品 → 使用者 → 產品 → 開發)

```
💻 需求不明確 → 👤 產品詢問使用者 → 👤 產品澄清 → 💻 開發實現
```

### 場景 4: Bug 報告 (測試 → 開發)

```
🧪 測試發現 bug → 💻 開發修復 → 🔍 審查重新檢查 → 🧪 測試驗證
```

### 場景 5: 知識積累 (開發 → FAQ)

```
💻 解決問題 → 📚 FAQ 代理記錄 → 未來: 快速獲得答案
```

## 💡 為什麼選擇多代理？

傳統的人工智慧助手獨自工作。Dream Creator 作為一個**團隊**工作：

1. **專業化**: 每個代理都是其領域的專家
2. **協作**: 代理之間相互溝通和交接任務
3. **升級機制**: 問題流向正確的專家
4. **知識共享**: 記錄解決方案供將來參考
5. **質量保證**: 多個代理審查和測試每個變更
